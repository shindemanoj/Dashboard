Introduction

Dashboard is an interactive reporting tool for visualization of stress test report data.

It automatically updates the instrument errors in real time.

Dashboard is lightweight MEAN stack(MongoDB, Express.js, AngularJS,Node.js) dynamic web application

User can select instrument type(GEM5K, GEM4K or GWP) from the dropdown which is located at rightmost corner. Dashboard consists of four main tabs, Overview, Graphs, Charts and Downloads.

Configuration

Dashboard contains properties file for each of three instruments in a Properties folder.


Figure A: Properties file structure

Properties file of an instrument looks similar to Figure A. It's in JSON format. User can add instrument details(Hostname/Network/SBC/Name) in InstConfig. User can add start date and end date in CommonConfig.properties file.

Code Structure

Dashboard follows MVC(Model View Controller) architecture.


Figure B: Client side code structure

Client side services are under services folder. Controller and HTML template for each page is under views folder.


Figure C: Server side code structure

Server side implementation is under project-server-code folder. Folder contains Server side APIs, database schema and database queries code.

API Documentation

  • Get All Reports
    • URL: "/api/dashboard/:instType"
    • Method: GET
    • URL Params : Required: instType=[String]
    • Response Codes: Success (200 OK), Bad Request (400)
  • Get all filenames of folder
    • URL: "/api/getfilenames/:path"
    • Method: GET
    • URL Params : Required: path=[String]
    • Response Codes: Success (200 OK), Bad Request (400)
  • Save or update the report
    • URL: "/api/dashboard"
    • Method: POST
    • Data Params : 
                      {
                  u : {
                    reportData: [JSON],
                    build: [String],
                    instType: [String],
                    overallFR: [Number],
                    stableFR: [Number],
                    unstableFR: [Number],
                    releaseData: [JSON],
                    startDate: [Date],
                    endDate: [Date]
                  }
                }
    • Response Codes: Success (200 OK), Bad Request (400)
