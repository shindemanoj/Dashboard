Introduction

Dashboard is a interactive reporting tool for visualization of stress test report data.

It automatically updates the instrument errors in real time.

Dashboard is lightweight MEAN stack(MongoDB, Express.js, AngularJS,Node.js) dynamic web application

User can select instrument type(GEM5K, GEM4K or GWP) from the dropdown. Dashboard consists of four main tabs, Overview, Graphs, Charts and Downloads.

Configuration

Dashboard contains properties file for each of three instruments in a Properties folder.

Properties file of an instrument looks similar to above image. It's in JSON format. User can add instrument details(Hostname/Network/SBC/Name) in InstConfig. User can add start date and end date in CommonConfig.properties file.

Code Structure

Dashboard follows MVC(Model View Controller) architecture.

Client side services are under services folder. Controller and HTML template for each page is under views folder.

Server side implementation is under project-server-code folder. Folder contains Server side APIs, database schema and database queries code.

API Documentation

  • app.get("/api/dashboard/:instType", findReports):
    • Get request to get all reports for a selected instrument.
  • app.get("/api/getfilenames/:path", getfilenames):
    • Get request to get all filenames from a instrument folder.
  • app.post("/api/dashboard", saveReport):
    • Post request save or update stress test report.
