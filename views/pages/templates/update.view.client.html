Steps to update Dashboard


1. Update build version in the instrument's properties file (Dashboard/Properties)
  • Every report of an instrument is mapped with the Version Number
  • Once you change the version number, Dashboard will automatically create new report in database
  • So, for every new build change version, start date and end date in properties file

Instrument Properties File

2. Update hostnames of instruments if changed.
  • Errors of an each instrument is counted with the help of it's hostname
  • If hostname of any instrument is changed update it in properties file

Instrument Properties File

3. Replace Error logs in the instrument's folder (Dashboard/instrument_name)
  • Replace old error log files with new one.
  • For correct parsing of files check for common errors in files
  • Like Extra commas or semicolons, header at 1st line, etc.
  • Remove those if present in any of the file
4. Go to Dashboard URL
  • Dashboard will automatically read all the changes in properties file and new error logs


How to start server of a Dashboard?


  • Dashboard server needs mongodb connection before it starts
  • Go to terminal and create mongodb connections (using "mongod" command)
  • Through terminal go to Dashboard folder and execute "node server.js" command
  • Access Dashboard url in browser
